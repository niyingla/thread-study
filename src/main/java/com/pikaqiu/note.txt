布隆过滤器
通过过滤条件和过滤位创建一个数组，通过散列函数将值映射到数组上，把位置置为1。
假定4个不同散列函数，传入一个字符串并进行一次插入操作，这时会进行4次散列，假设到了4个不同的下标，这个时候我们就会去数组中，将这些下标的位置置为1。
如果接下来我们再传入同一个字符串时，因为4次的散列结果都是跟上一次一样的，这个时候我们就可以认为这个字符串是已经存在的了。

cap理论 最多三选二
 一致性 C 总是能读到最新数据 数据库一般 CA
 可用性 A 所有请求都要收到一个非错误返回 （和一致性冲突部分 数据同步节点需要时间 多个写入点就必须选一个）
 分区容错性 P 多个服务单元之间，可以容错。 一般难兼得

 网络七层 （实际只 运用了五层）
 应用层 http,ping 网络应用怎么写，为应用提供网络能力。
 传输层 例如 tcp,udp,网络出发到到达的ip、端口，提供应用之间的连接能力。
 网络层 ip协议 终端到终端的发送。 (手机，服务器)
 链接层 数据交换。
 物理层 设备传输链路。

 tcp 数据包发送
 一次响应返回多个数据包，一次性响应多个包，接受数据后按序组合成完整响应，缺失包根据序号补发。

 滑动窗口
 三个段 已接收 发送中（还未响应接受成功） 未发送
 一般来说窗口大小=5，单个响应成功后窗口会右移，丢失某个后整个窗口都会重发。

HTTPS
1 客户端获取证书和公玥，进行验证，获取解密后的摘要信息，进行比较。（RSA 非对称加密）
2 成功后通过public key 加密并发送 hello+随机摘译
3 服务端 私钥解码成功 后返回ack
4 后续都通过随机摘要 进行AES对称加密通讯

HTTP2
1 二进制传输（文本-》 二进制 优化网络带宽占用）
2 多路复用 （一次可以发送请求多个）
3 服务端推送 （例如 加载html 会主动推送css，js）
